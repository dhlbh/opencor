Introduction

Context

The development and exchange of a mathematical model usually requires several steps:

  • Write some model ideas on paper;
  • Implement these ideas in C++, Fortran, MATLAB, etc.;
  • Write and submit a manuscript which describes the model and includes its mathematical formulations;
  • The manuscript gets accepted and published; and
  • The manuscript is read and the model implemented by one or several people.
However, each of these steps is subject to human error.

CellML

To address the above problem, Prof. Peter Hunter's group (Auckland, New Zealand) has, in August 2001, released version 1.0 of the specifications of the CellML language, before releasing a refined version (1.1) in February 2006.

CellML supports the definition and sharing of models through an XML-based language, and provides a consistent mathematical representation of a model through MathML. CellML models are thus independent of the operating system or programming language used by the user, and model equations can be generated and published from the CellML code. Model evolution and re-use are strongly encouraged (and are possible using CellML 1.1).

OpenCOR

OpenCOR is a cross-platform CellML-based modelling environment which has, from its inception, been developed with modellers, experimentalists and teachers/students in mind. It tries to alleviate all of the complexity associated with CellML by providing a user-friendly way to organise, edit, simulate and analyse CellML files.
