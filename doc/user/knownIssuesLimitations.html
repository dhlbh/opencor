Known Issues/Limitations

Please note that the following known issues/limitations have been observed on Windows 7, Ubuntu 10.10 (Maverick Meerkat; both 32-bit and 64-bit) and Mac OS X 10.6 (Snow Leopard). So, if you use a different operating system, these issues/limitations may or not be relevant to you.

Windows, Linux and Mac OS X

By default, OpenCOR uses your system's language for menus, dialog boxes, etc., as long as it is either English or French (please contact us if you would like another language to be added). If your system uses another language, then OpenCOR will default to English. Otherwise, if you specifically specify English or French as the language to be used by OpenCOR, then please be aware that system messages, dialog boxes, etc. will be displayed in your system's language.

Windows

OpenCOR's embedded file browser may, on some computers, result in OpenCOR being slow to respond at startup. Please note that this has nothing to do with OpenCOR, but with Windows and is a well-known problem for which there are various causes and solutions.

Linux

Depending on your window manager (e.g. Compiz), OpenCOR may or not experience one of the following issues/limitations (none of which affect the good functioning of the software itself):

  • Upon starting OpenCOR, the software will try to retrieve your previous session's settings which may result in the main window showing up lower down the screen than it should; and
  • To show the help window by pressing the F1 key may, if the help window is to be floating (as opposed to being docked to the main window), result in OpenCOR not being the active application anymore. This means that if you were to press the F1 key one more time, the help window will remain visible while it should have disappeared.

Note that the above issues have nothing to do with OpenCOR itself (everything works fine on both Windows and Mac OS X), but with the window manager that you use. Indeed, there is a lack of compliancy between the various window managers, so something that works with one window manager may or not work with another, as is unfortunately the case with OpenCOR.

Another non window manager related issue/limitation is:

  • Menu items that have sub-menus will not, by default, display their icon (should they have one). This has nothing to do with OpenCOR, but is instead the expected behaviour on Ubuntu. As mentioned here, all you need to do to change this behaviour is to enter the following at the command line: 
    $ gconftool-2 --type Boolean --set /desktop/gnome/interface/menus_have_icons True
