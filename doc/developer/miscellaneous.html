Ideas Worth Exploring/Implementing

Should any of the ideas below ever get implemented, then they should then be documented and put somewhere under the user documentation.

  • For various reasons (test suites, parameter sweep, etc.), it would be good for OpenCOR to also work as a command line tool, and have it run all the simulations (or whatever is required of it), and then exit. This might require an input file of sorts, which may or not prove practical. For example: 
    $ ./OpenCOR -i myInputFile
    Indeed, in some cases, it would probably be more convenient if one could just pass some arguments to OpenCOR to, say, change the default value of this or that parameter (rather than having to write another input file). In that context, maybe one could have some XML file that describes a particular set of command line options (together with corresponding actions, using some kind of a scripting language?) which would be passed together with the 'normal' command line options. For example, we could have something like: 
    $ ./OpenCOR -s mySyntaxFile -a -c 1 -o myOutputFile
    where mySyntaxFile would be the aforementioned XML file and the rest various flags/options.
